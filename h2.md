## a) Sudoless
I started by creating a new user _kivimies_ and the group _sudoless_ mentioned in the assignment. And then I added the user _kivimies_ into the group _sudoless_. I'm not sure if I did the next part right, but I switched users on my local machine and created a new ssh key-pair and copied the public key to _localhost_ to enable login without password. 

<img width="802" height="233" alt="image" src="https://github.com/user-attachments/assets/53147857-b651-4546-b9e1-0f5ffb016f33" />

Then I had to create a new sudoers rule for the _sudoless_ group so that users in that groupt could use sudo without a password. The rule was made in to the _/etc/sudoers.d/sudoless_ file, and if you are wondering what the heck is _sudoers.d_ directory, you are at the same point as I was 15 minutes before writing this. So it's a directory that contains extra sudo rules separeted from the main sudo config file _/etc/sudoers_. I read some articles and Red Hat documentations and I noticed that main difference between these two (/etc/sudoers, /etc/sudoers.d/) is in the maintainability of the rules in them. I will not go deeper into this subject since I don't know much about it so I will leave the links to the atricles etc. I just read, [RedHat](https://docs.redhat.com/en/documentation/red_hat_enterprise_linux/8/html/configuring_basic_system_settings/managing-sudo-access_configuring-basic-system-settings), [unix.stackexchange](https://unix.stackexchange.com/questions/375433/etc-sudoers-vs-etc-sudoers-d-file-for-enabling-sudo-for-a-user), [superuser](https://superuser.com/questions/869144/why-does-the-system-have-etc-sudoers-d-how-should-i-edit-it).

But let's get to the creating of the sudoers rule. This was easy since it was just the one line. Just remember to use Visudo to make it.
