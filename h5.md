[1.3 Getting Started - What is Git?](https://git-scm.com/book/en/v2/Getting-Started-What-is-Git%3F) 

Git is a version control system (VCS) that lets you save snapshots of your projects on your local machine and later publish them to a remote repository (like GitHub). And what are snapshots? They are complete versions of your projects created every time you run `git commit`. This makes it easy to try new approaches and ideas because you can always go back to any sanpshot.
Git works locally so you don't need to internet to code or commit changes because these are saves locally on your machine. And once you're online you can just use `git push` to upload them to remote repo.

[git add --all && git commit; git pull && git push ](https://terokarvinen.com/palvelinten-hallinta/#h5-gitar-hero)  

Git is primarily used with the command mentioned above so let's go through what they mean. `git add -all` means that you "stage all changes in the working directory so they can be included in the next commit" (geeksforgeeks,2016). And obviously `git commit` saves the snapshot of the project locally. 
'git pull && git push' are used when updating your local/remote repository. `git pull` pulls changes from the remote repository to your local repo and `git push` pushes local changes to remote repo.
